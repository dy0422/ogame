# Native macOS OGame 剩余计划总表

当前状态：Milestone 1-6 已形成可玩的 SwiftUI macOS 单机版；Milestone 7 已启动，完成了服务端图片接入、指挥官简报、胜利结算、平衡工具和基础打包。

## Milestone 7：主动 AI 对手与开局体验

- AI 造舰：矿工、防御型、掠袭者、扩张者按策略建造舰船、防御和探测器。
- AI 派遣：侦察、运输、攻击、殖民、回收残骸全部由 AI 自主发起。
- AI 情报限制：AI 只能用已侦察情报、公开排名、关系记忆，不读玩家隐藏库存。
- AI 难度：简单、标准、困难对应不同侦察频率、攻击阈值、防御权重和扩张欲望。
- 离线 AI 上限：离线补算时限制连续攻击和事件爆炸，避免打开游戏后被瞬间清空。
- 新手前 15 分钟：加入任务链、推荐动作、第一次升级、第一次造船、第一次侦察、第一次出征。
- 局后体验：胜利/失败复盘、关键时间点、最佳时间记录、重新开局保留节奏反馈。

## Milestone 8：内容深度与 OGame 复刻

- 科技门槛：舰船、防御、建筑、研究统一走规则需求，UI 显示锁定原因。
- 建筑深度：补齐更完整的仓库、纳米工厂、导弹发射井、月球设施。
- 生产控制：矿场百分比调节、电力影响预览、停产/降产策略。
- 加速机制：机器人、纳米、研究实验室等级影响建造和研究时间。
- 导弹系统：星际导弹、拦截导弹、导弹井容量、目标防御选择、战报展示。
- 月球系统：月球建筑、感应阵、跳跃门、月球形成概率、月球 UI。
- 舰船扩展：补轰炸机、毁灭者、死星、战斗巡洋舰、太阳能卫星等后期单位。
- 战斗复刻：更接近 OGame 的攻防盾、快速射击、回合战斗和残骸规则。
- 胜利路线扩展：经济、科技、统治、探索加入后期目标，不只看短线分数。

## Milestone 9：平衡实验室与快节奏单机局

- 场景模拟器：固定 seed 跑 30/60/120/240 分钟，输出首舰、首侦察、首战斗、首殖民、胜利时间。
- 目标窗口：首舰 10-25 分钟，首出征 20-45 分钟，首冲突 45-90 分钟，胜利 2-4 小时。
- 基线测试：把平衡工具输出变成回归测试，避免后续改动破坏节奏。
- AI 曲线调参：调整 AI 建造间隔、攻击阈值、扩张速度、防御反应。
- 经济曲线调参：矿产、仓库、电力、造船成本、燃料成本、科研成本重新平衡。
- 玩家节奏补偿：开局任务奖励、首次舰队奖励、探索奖励、失败缓冲。
- 离线节奏调参：正常离线有收益，但不能让 AI 在离线期间过度滚雪球。

## Milestone 10：UI 图片复刻、SwiftUI 拆分与产品打磨

- UI 拆分：把 `ContentView.swift` 拆成 Dashboard、Planet、Fleet、Strategy、Research、Settings、Shared。
- AppModel 拆分：把存档、舰队、经济、科研、战报、胜利、简报拆成更小的 presentation/service 文件。
- 服务端图片深化：战报、星图、舰队派遣、胜利页继续按 `skins/xnova` 优化图像密度。
- 原版风格增强：建筑/科技/舰船详情图、资源图标、星球背景、残骸/月球视觉统一。
- 帮助系统：玩家指南、首局提示、规则说明、胜利路线说明、存档说明。
- macOS 体验：菜单命令、快捷键、设置入口、窗口尺寸、空状态、按钮可访问性。
- 视觉与音效：应用图标、基础音效、事件提示音、轻量动效。
- 中文完善：所有战报、系统事件、按钮、错误提示、导出说明统一中文。

## Milestone 11：存档、迁移与发行硬化

- 存档迁移：从旧 schema 自动迁移到当前 schema，未来版本拒绝覆盖。
- 存档完整性：损坏存档检测、备份验证、失败时禁止覆盖 autosave。
- 导入导出：手动导出存档、导入存档、备份重命名、删除保护。
- 发布脚本：统一 `verify-release.sh`，串联测试、构建、打包、资源检查。
- macOS 包：生成 `.app`、`.zip`、后续可选 `.dmg`。
- 签名与公证：开发者证书存在时自动签名；没有证书时输出手动说明。
- 版本信息：版本号、构建号、发布说明、已知简化项。

## Milestone 12：Release Candidate 与最终验收

- 手动完整局：至少打一局新档，记录关键时间和卡点。
- 回归修复：每个 release blocker 先补测试再修。
- 性能检查：长时间模拟、离线补算、报告数量、事件数量、UI 列表不卡顿。
- 保存/读取检查：新档、旧档、备份、导入、损坏文件全部走一遍。
- 最终清单：文档、包体、测试、资源、GitHub 推送全部确认。
- RC 标签：创建 `native-ogame-v1-rc1`，推送 tag。

## 推荐执行顺序

1. Milestone 7：主动 AI 与新手体验。
2. Milestone 9：平衡实验室，因为 AI 做完后必须先校准。
3. Milestone 8：内容深度和 OGame 复刻。
4. Milestone 10：UI 拆分、图片深化和产品打磨。
5. Milestone 11：存档迁移和发行硬化。
6. Milestone 12：完整 playtest、修 bug、打 RC。

## 最终完成标准

- 新档不用改数据也能经历升级、造舰、侦察、战斗、殖民、回收、胜利。
- AI 能主动发展、侦察、攻击、扩张、回收，但不读取隐藏信息。
- 快节奏单机局能在 2-4 小时模拟时间内产生清晰胜负。
- 服务端图片资源在主要 UI 中都被合理使用。
- 存档能迁移、备份、导入、导出、损坏保护。
- `bash scripts/verify-release.sh` 通过。
- 能生成可交付给普通 Mac 用户打开的发布包。
