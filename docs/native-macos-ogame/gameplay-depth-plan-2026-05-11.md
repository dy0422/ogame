# 玩法深度 1-5 执行说明

本轮目标是把“游戏性审计”的 1-5 项从建议变成可运行系统，重点不是再加一批静态描述，而是让新档可以被真实模拟、被顾问解释、被路线检查点追踪。

## 1. 真实平衡审计

- 新增 `GameplayAuditEngine.runAutoplayAudit`。
- 审计不使用 `BalanceScenarioRunner` 的引导夹具，不直接塞资源、舰船、月球或胜利。
- 推进来源只有舰队优先自动升级、顾问驱动的基础舰队动作、AI 策略和常规模拟 tick。
- `OGameBalanceTool` 追加 `Autoplay gameplay audit` 输出，用于观察真实开局是否自然出现造舰、出航、探索、殖民、战斗和顾问覆盖。
- 自动升级新增第一艘船预算保护：开局补齐最低矿电骨架后，会给首艘舰船留资源窗口，避免无限升级矿场导致舰队循环永远不开。
- 快节奏单机新档初始资源调整为 `2,000 / 2,000 / 500`，保证真实开局和自动托管不再完全依赖测试夹具进入造舰阶段。

## 2. 胜利路线检查点

- 新增 `VictoryRoutePlanEngine`。
- 四条路线从单一数字胜利扩展成检查点：
  - 经济：经济体量、殖民网络、能源稳定、殖民分工。
  - 科技：科研总量、战斗科技、引擎体系、月面设施。
  - 统治：星域占比、主力舰队、防线成型、战斗经验。
  - 探索：星图覆盖、远征循环、残骸利用、外域落点。
- `StrategicAdvisorEngine` 会把最接近完成的路线作为“路线建议”展示。

## 3. AI 意图链

- 新增 `AIIntentEngine`。
- AI 会被归类为建设、侦察、扩张、掠袭、回收、防御或闲置。
- 意图来源包括在途舰队、策略类型、舰队库存、威胁记忆和残骸线索。
- 顾问新增 `aiThreat` 类型，用来提示玩家哪个 AI 正在形成压力。

## 4. 中后期玩家目标

- `PlayerObjectiveKind` 新增殖民分工、战报复盘、舰队保存演练、跳跃门网络。
- `PlayerObjectiveEngine` 会根据殖民专精、战斗报告、召回舰队和月球跳跃门统计进度。
- 这些目标补足了从“能造能打”到“会运营会避险”的中期学习路径。

## 5. 战略顾问扩展

- 顾问新增 `victoryRoute`、`aiThreat`、`combatReview`。
- 除经济、队列、残骸、殖民和远征外，顾问现在能提示：
  - 当前最值得推进的胜利路线。
  - 可能威胁玩家的 AI 动向。
  - 最近战斗是否值得复盘。

## 验证方式

```bash
swift run OGameCoreTests
swift run OGameBalanceTool
```

核心判断：

- 单元测试覆盖真实审计、路线检查点、AI 意图、中后期目标和顾问新类型。
- 平衡工具中 `used_guided_fixtures` 应保持 `false`。
- 真实自动审计应能在 4 小时快节奏窗口内看到 `first_ship` 和 `first_fleet`。
- `route_progress` 不应只被资源囤积瞬间打满，殖民、能源、月球、战斗经验等检查点会继续约束路线完成度。
