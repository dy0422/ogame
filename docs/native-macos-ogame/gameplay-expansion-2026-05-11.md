# 三阶段玩法扩展说明

本轮目标是把单机版从“资源和队列推进”继续推向“星区机会、行动链、长期局势”三层循环。新增内容都落在 `OGameCore`，会随正常模拟 tick 自动刷新，并通过战略顾问先进入 UI。

## 阶段一：星区事件、PVE 目标、行动链

- 星区事件：周期生成海盗活动、远古遗迹等机会，带有坐标、持续时间、收益修正和风险修正。
- PVE 敌对目标：生成海盗补给站、外星前哨等非玩家目标，提供威胁等级、建议战力和资源奖励。
- 行动链：把“侦察目标 -> 打击据点 -> 回收战利品”串成连续任务，让舰队、探测器和回收船都有明确用途。

这一层解决短中期“下一步打哪里”的问题，适合快节奏单机局里不断给玩家制造小目标。

## 阶段二：星区控制、贸易线、深度情报

- 星区控制：同一势力在同星系拥有多个星球时形成控制等级，给资源和传感加成预留规则入口。
- 贸易路线：在同势力星球之间生成补给线建议，显示资源流量和风险，用来连接殖民地经济。
- 深度情报：基于间谍等级生成信号截获行动，提示 AI 意图、舰队倾向和反侦察风险。

这一层让殖民选址、星球集群、情报科技不只是数值增长，而是形成可读的战略版图。

## 阶段三：舰队 Doctrine、遗物、后期危机

- 舰队编组 doctrine：根据玩家船只给出突袭、远征、补给等编组倾向，补充舰种搭配目标。
- 遗物：通过远古遗迹、战斗报告、探索记录解锁蓝图或档案，作为长期成长收集项。
- 后期危机：宇宙成熟后触发海盗王集结，提供持续升级的风险和清剿压力。

这一层服务后期目标，把胜利前后的空档变成可持续处理的局势。

## 验证口径

- `OGameCoreTests` 覆盖旧存档默认值、三阶段状态生成和战略顾问露出。
- `OGameBalanceTool` 的 `Autoplay gameplay audit` 增加 `expansion_signals` 字段，用来观察自动推进中新增玩法信号是否自然出现。
- 新字段都支持旧 JSON 缺省解码，避免已有存档因为缺少扩展状态而无法读取。
